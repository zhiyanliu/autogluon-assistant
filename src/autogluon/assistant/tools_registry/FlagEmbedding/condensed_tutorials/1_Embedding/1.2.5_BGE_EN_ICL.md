## Architecture

BGE-EN-ICL uses a **decoder-only LLM (Mistral-7B)** as its base, unlike previous encoder-only BGE models.

```python
from transformers import AutoTokenizer, AutoModel
tokenizer = AutoTokenizer.from_pretrained("BAAI/bge-en-icl")
raw_model = AutoModel.from_pretrained("BAAI/bge-en-icl")
```

## Last Token (EOS) Pooling

Instead of replacing unidirectional attention with bidirectional (which creates a mismatch with pre-training), BGE-EN-ICL uses the **[EOS] token's output embedding** as the text representation: $h_t = \text{LLM}(T)[\text{EOS}]$

```python
def last_token_pool(last_hidden_states: torch.Tensor,
                    attention_mask: torch.Tensor) -> torch.Tensor:
    left_padding = (attention_mask[:, -1].sum() == attention_mask.shape[0])
    if left_padding:
        return last_hidden_states[:, -1]
    else:
        sequence_lengths = attention_mask.sum(dim=1) - 1
        batch_size = last_hidden_states.shape[0]
        return last_hidden_states[torch.arange(batch_size, device=last_hidden_states.device), sequence_lengths]
```

## In-Context Learning (Few-Shot)

BGE-EN-ICL supports both **zero-shot** (same as BGE v1/1.5) and **few-shot** inference via ICL examples.

Few-shot examples format — each with `instruct`, `query`, and `response`:

```python
examples = [
    {
        'instruct': 'Given a web search query, retrieve relevant passages that answer the query.',
        'query': 'what is a virtual interface',
        'response': "A virtual interface is a software-defined abstraction..."
    },
    {
        'instruct': 'Given a web search query, retrieve relevant passages that answer the query.',
        'query': 'causes of back pain in female for a week',
        'response': "Back pain in females lasting a week can stem from various factors..."
    }
]
```

**Using `FlagICLModel`:**

```python
from FlagEmbedding import FlagICLModel

model = FlagICLModel(
    'BAAI/bge-en-icl',
    examples_for_task=examples,          # set None for zero-shot
    examples_instruction_format="<instruct>{}\n<query>{}\n<response>{}",
    devices=[0],
)

embeddings_1 = model.encode_queries(queries)
embeddings_2 = model.encode_corpus(documents)
similarity = embeddings_1 @ embeddings_2.T
```

**Key details:**
- `examples_for_task=None` → zero-shot mode
- `examples_instruction_format` controls how ICL examples are formatted into the prompt
- Queries and corpus are encoded separately via `encode_queries` / `encode_corpus`
- Similarity computed via dot product
