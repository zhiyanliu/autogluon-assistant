# BGE Auto Embedder - Condensed Tutorial

`FlagAutoModel` from FlagEmbedding provides unified inference for BGE and other open-source embedding models (E5, GTE, SFR, etc.).

```bash
pip install FlagEmbedding
```

## Usage

```python
from FlagEmbedding import FlagAutoModel

model = FlagAutoModel.from_finetuned(
    'BAAI/bge-base-en-v1.5',
    query_instruction_for_retrieval="Represent this sentence for searching relevant passages: ",
    devices="cuda:0",   # uses all available GPUs or CPU if not specified
)

queries = ["query 1", "query 2"]
corpus = ["passage 1", "passage 2"]

q_embeddings = model.encode_queries(queries)
p_embeddings = model.encode_corpus(corpus)
scores = q_embeddings @ p_embeddings.T
```

The API mirrors the underlying model class (`FlagModel`, `FlagM3Model` for BGE M3, `FlagLLMModel` for BGE Multilingual Gemma2, `FlagICLModel` for BGE ICL).

## Model Mapping

`FlagAutoModel` uses an `OrderedDict` called `AUTO_EMBEDDER_MAPPING` to store supported model configurations, each as an `EmbedderConfig`:

```python
@dataclass
class EmbedderConfig:
    model_class: Type[AbsEmbedder]
    pooling_method: PoolingMethod
    trust_remote_code: bool = False
    query_instruction_format: str = "{}{}"
```

## Adding Custom Models

To register a custom model in `FlagAutoModel`, determine these four attributes:

1. **Model class** — encoder or decoder-based?
2. **Pooling method** — CLS token, mean pooling, or last token?
3. **`trust_remote_code`** — required for some custom architectures
4. **Query instruction format** — retrieval-specific formatting

Then add your model name as key and its `EmbedderConfig` as value to `MODEL_MAPPING`.
