# Similarity Metrics — Condensed Tutorial

## 1. Jaccard Similarity

Measures overlap between word sets: $J(A,B)=\frac{|A\cap B|}{|A\cup B|}$, range $[0, 1]$.

```python
def jaccard_similarity(sentence1, sentence2):
    set1 = set(sentence1.split(" "))
    set2 = set(sentence2.split(" "))
    return len(set1.intersection(set2)) / len(set1.union(set2))
```

**Limitation:** Purely lexical — ignores word order and semantics. Sentences sharing words but with different meanings score high, while paraphrases score low.

## 2. Euclidean Distance

$d(A, B) = \sqrt{\sum_{i=1}^n (A_i-B_i)^2}$, range $[0, +\infty)$. Lower = more similar.

```python
# Manual
dist = torch.sqrt(torch.sum(torch.pow(torch.subtract(A, B), 2), dim=-1))
# Built-in
torch.cdist(A, B, p=2).item()
```

## 3. Cosine Similarity

$\cos(\theta)=\frac{A\cdot B}{\|A\|\|B\|}$, range $[-1, 1]$. Higher = more similar.

**⚠️ Naive computation** (dot product / product of norms) can lose precision and produce values > 1.0. **PyTorch's approach** normalizes first to avoid this:

```python
# PyTorch-style (more numerically stable)
res = torch.mm(A / A.norm(dim=1), B.T / B.norm(dim=1))

# Recommended: use built-in function
import torch.nn.functional as F
F.cosine_similarity(A, B).item()
```

## 4. Dot Product

$$A\cdot B = \sum_{i=1}^{n}A_i B_i = \|A\|\|B\|\cos(\theta)$$

```python
dot_prod = (A @ B.T).item()
```

### Dot Product vs. Cosine Similarity

- **Cosine similarity**: only considers angle (normalized by magnitude)
- **Dot product**: considers both magnitude and angle

**Key detail:** BGE models normalize output embeddings to magnitude 1, so dot product and cosine similarity yield identical results.

```python
from FlagEmbedding import FlagModel
model = FlagModel('BAAI/bge-large-en-v1.5',
                  query_instruction_for_retrieval="Represent this sentence for searching relevant passages:",
                  use_fp16=True)
embedding = torch.tensor(model.encode("I am very interested in NLP"))
torch.norm(embedding).item()  # ≈ 1.0
```

## 5. Practical Example — Why Embeddings Matter

```python
sentence_1 = "I will watch a show tonight"
sentence_2 = "I will show you my watch tonight"      # same words, different meaning
sentence_3 = "I'm going to enjoy a performance this evening"  # paraphrase of s1
```

| Metric | s1 vs s2 | s1 vs s3 | Correct? |
|--------|----------|----------|----------|
| Jaccard | 0.625 (high) | 0.077 (low) | ❌ |
| Euclidean (embedding) | larger | smaller | ✅ |
| Cosine (embedding) | lower | higher | ✅ |

```python
embeddings = torch.from_numpy(model.encode([sentence_1, sentence_2, sentence_3]))
embedding_1, embedding_2, embedding_3 = [embeddings[i].view(1, -1) for i in range(3)]

# Euclidean
torch.cdist(embedding_1, embedding_2, p=2).item()  # larger distance
torch.cdist(embedding_1, embedding_3, p=2).item()  # smaller distance

# Cosine
F.cosine_similarity(embedding_1, embedding_2).item()  # lower similarity
F.cosine_similarity(embedding_1, embedding_3).item()  # higher similarity
```

**Takeaway:** Embedding-based metrics (Euclidean, cosine) capture semantic meaning, correctly identifying paraphrases and distinguishing sentences that share words but differ in meaning — unlike lexical methods like Jaccard.
