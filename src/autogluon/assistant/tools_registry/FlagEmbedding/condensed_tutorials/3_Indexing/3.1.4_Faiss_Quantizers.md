# Faiss Quantizers

## Setup

```python
import faiss
import numpy as np
np.random.seed(768)
data = np.random.random((1000, 128))
```

## 1. Scalar Quantizer

Transforms 32-bit float vectors to lower precision (e.g., 8-bit int) for ~4x size reduction. Each dimension is distributed into 2^qtype buckets.

**Three classes:** `ScalarQuantizer` (quantizer), `IndexScalarQuantizer` (flat index), `IndexIVFScalarQuantizer` (IVF index). Key params: `d` (dimension), `qtype` (bit depth), `metric` (L2/IP), `nlist` (IVF clusters).

```python
d, k = 128, 3
qtype = faiss.ScalarQuantizer.QT_8bit

# Standalone quantizer
quantizer = faiss.ScalarQuantizer(d, qtype)
quantizer.train(data)
new_data = quantizer.compute_codes(data)

# As index (flat or IVF)
index = faiss.IndexScalarQuantizer(d, qtype, faiss.METRIC_L2)
# index = faiss.IndexIVFScalarQuantizer(d, nlist, qtype, faiss.METRIC_L2)
index.train(data)
index.add(data)
D, I = index.search(data[:1], k)
```

## 2. Product Quantizer

Splits vectors of dimension `d` into `M` sub-vectors (d/M each), clusters each subspace into 2^nbits centroids, then stores only centroid indices. Distances are computed against codebook centroids for fast approximate search.

**Three classes:** `ProductQuantizer`, `IndexPQ`, `IndexIVFPQ`. Key params: `d`, `M` (sub-vectors, **d % M must == 0**), `nbits` (bits per subquantizer), `metric`.

```python
d, M, nbits, k = 128, 8, 4, 3

# Standalone quantizer
quantizer = faiss.ProductQuantizer(d, M, nbits)
quantizer.train(data)
new_data = quantizer.compute_codes(data)

# Flat PQ index
index = faiss.IndexPQ(d, M, nbits, faiss.METRIC_L2)
index.train(data)
index.add(data)
D, I = index.search(data[:1], k)

# IVF PQ index (requires a coarse quantizer)
nlist = 5
coarse_quantizer = faiss.IndexFlat(d, faiss.METRIC_L2)
index = faiss.IndexIVFPQ(coarse_quantizer, d, nlist, M, nbits, faiss.METRIC_L2)
index.train(data)
index.add(data)
D, I = index.search(data[:1], k)
```

**Key distinctions:** Quantizer classes compress data manually (`train` → `compute_codes`). Index classes handle quantization automatically on `add()`. IVF variants (`IndexIVFScalarQuantizer`, `IndexIVFPQ`) add inverted file partitioning for faster search on large datasets; `IndexIVFPQ` requires a separate coarse quantizer (e.g., `IndexFlat`).
