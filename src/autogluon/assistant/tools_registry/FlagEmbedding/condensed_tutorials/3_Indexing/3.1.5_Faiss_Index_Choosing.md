# Condensed: Choosing Index

Summary: This tutorial demonstrates how to implement and compare six Faiss ANN index types—Flat (IndexFlatL2), IVF (IndexIVFFlat), HNSW (IndexHNSWFlat), LSH (IndexLSH), Scalar Quantizer (IndexScalarQuantizer), and Product Quantizer (IndexPQ)—covering construction, parameter configuration (nlist/nprobe, M/efSearch/efConstruction, nbits, QT_8bit), training, adding vectors, and searching. It includes a recall computation function for benchmarking, dataset loading from HDF5, and a speed/recall/memory tradeoff summary. Useful for tasks involving vector similarity search setup, index selection, and ANN system benchmarking with Faiss.

*This is a condensed version that preserves essential implementation details and context.*

# Choosing a Faiss Index

## Setup

```python
from urllib.request import urlretrieve
import h5py, faiss, numpy as np

# Download SIFT1M dataset (1M vectors, 128-dim, L2 distance)
urlretrieve("http://ann-benchmarks.com/sift-128-euclidean.hdf5", "data.hdf5")

with h5py.File('data.hdf5', 'r') as f:
    corpus, query = f['train'][:], f['test'][:]

d = corpus[0].shape[0]  # 128
k = 100

def compute_recall(res, truth):
    recall = 0
    for i in range(len(res)):
        recall += len(np.intersect1d(res[i], truth[i])) / len(res[i])
    return recall / len(res)
```

## 1. Flat Index (Ground Truth Baseline)
Brute-force search — **100% recall**, slowest.

```python
index = faiss.IndexFlatL2(d)
index.add(corpus)
D, I_truth = index.search(query, k)
```

## 2. IVF Index
Partitions space into Voronoi cells; searches only `nprobe` nearest cells.

```python
nlist, nprob = 5, 3
quantizer = faiss.IndexFlatL2(d)
index = faiss.IndexIVFFlat(quantizer, d, nlist)
index.nprobe = nprob
index.train(corpus)
index.add(corpus)
```
**Result:** Significant speed improvement with minimal recall loss.

## 3. HNSW Index
Graph-based — extremely fast search, but **slower index construction and larger memory footprint**.

```python
M, ef_search, ef_construction = 64, 32, 64
index = faiss.IndexHNSWFlat(d, M)
# Must set BEFORE adding data
index.hnsw.efConstruction = ef_construction
index.hnsw.efSearch = ef_search
index.add(corpus)  # no train() needed
```
**Result:** Sub-second search, acceptable recall reduction. Best for extreme search speed when memory allows.

## 4. LSH
⚠️ **Not recommended for high-dimensional data.** At dim=128, even with `nbits = d * 8`, both index creation and search are slow, with poor recall (~58.6%).

```python
index = faiss.IndexLSH(d, d * 8)
index.train(corpus)
index.add(corpus)
```

## 5. Scalar Quantizer Index
Quantizes each float to fewer bits (e.g., 8-bit). Performance depends on data distribution.

```python
index = faiss.IndexScalarQuantizer(d, faiss.ScalarQuantizer.QT_8bit, faiss.METRIC_L2)
index.train(corpus)
index.add(corpus)
```
**Note:** On SIFT (integer values in [0, 218]), recall is nearly identical to Flat. Difference is more pronounced on complex float32 distributions.

## 6. Product Quantizer Index
Splits vectors into `M` sub-vectors, quantizes each to `nbits`. Balanced tradeoffs — commonly **combined with IVF or HNSW** in production.

```python
index = faiss.IndexPQ(d, M=16, nbits=8, metric=faiss.METRIC_L2)
index.train(corpus)
index.add(corpus)
```

## Summary

| Index | Speed | Recall | Memory | Notes |
|-------|-------|--------|--------|-------|
| **Flat** | Slowest | 100% | High | Ground truth baseline |
| **IVF** | Fast | Very high | Moderate | Tune `nlist`/`nprobe` |
| **HNSW** | Fastest search | High | **Largest** | Slow build; set params before `add()` |
| **LSH** | Slow | Poor (~59%) | High | Avoid for high-dim data |
| **Scalar Quantizer** | Fast | ~Flat (data-dependent) | Reduced | Best when value range is limited |
| **Product Quantizer** | Moderate | Moderate | **Smallest** | Best combined with IVF/HNSW |
