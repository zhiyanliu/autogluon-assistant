## Embedding & Indexing Pipeline

```python
from FlagEmbedding import FlagModel
import numpy as np
import faiss

# Encode corpus
model = FlagModel('BAAI/bge-base-en-v1.5',
                  query_instruction_for_retrieval="Represent this sentence for searching relevant passages:",
                  use_fp16=True)
corpus_embeddings = model.encode(corpus)
```

> **⚠️ Faiss only accepts float32 inputs.** Convert before indexing:

```python
corpus_embeddings = corpus_embeddings.astype(np.float32)
```

### Build Index

```python
dim = corpus_embeddings.shape[-1]  # 768 for bge-base-en-v1.5
index = faiss.index_factory(dim, 'Flat', faiss.METRIC_INNER_PRODUCT)

# For faiss-gpu:
# co = faiss.GpuMultipleClonerOptions()
# index = faiss.index_cpu_to_all_gpus(index, co)

# No training needed for "Flat" quantizer; quantization-based indices may require index.train()
index.add(corpus_embeddings)
```

### Save/Load Index

```python
faiss.write_index(index, "./index.bin")
index = faiss.read_index("./index.bin")
```

## Search

```python
query_embeddings = model.encode_queries(queries)
dists, ids = index.search(query_embeddings, k=3)

for i, q in enumerate(queries):
    print(f"query:\t{q}\nanswer:\t{corpus[ids[i][0]]}\n")
```

**Key points:**
- `encode_queries()` for queries vs `encode()` for corpus (applies query instruction automatically)
- `index.search()` returns distances and indices for top-k nearest neighbors
- `Flat` index = exact search; other index types (e.g., IVF, PQ) trade accuracy for speed on large datasets but may require `index.train()`
