# Evaluate Reranker (Condensed)

## Key Concepts

Rerankers capture latent semantic meanings better than embedders but have **O(N²) complexity**. Common use: rerank top-k results from an embedding model. Evaluation compares how well rerankers improve candidates from a base embedder (here, `bge-large-en-v1.5`) on BEIR benchmark.

> **Warning:** Requires GPU. Pipeline is very time-consuming. Example uses single BEIR task (FiQA).

## Evaluation Pipeline

Both rerankers use the same command structure via `FlagEmbedding.evaluation.beir`. **Critical parameters:**

| Parameter | Value | Notes |
|---|---|---|
| `search_top_k` | 1000 | Initial embedding retrieval |
| `rerank_top_k` | 100 | Candidates passed to reranker |
| `eval_metrics` | ndcg_at_10, recall_at_100 | |
| `embedder_name_or_path` | BAAI/bge-large-en-v1.5 | Base embedder (same for both) |
| `ignore_identical_ids` | True | |

**Model 1 — bge-reranker-large** (~560M params, BERT-like):
```bash
python -m FlagEmbedding.evaluation.beir \
  --eval_name beir --dataset_dir ./beir/data --dataset_names fiqa \
  --splits test dev --corpus_embd_save_dir ./beir/corpus_embd \
  --output_dir ./beir/search_results --search_top_k 1000 --rerank_top_k 100 \
  --overwrite True --k_values 10 100 --eval_output_method markdown \
  --eval_output_path ./beir/beir_eval_results.md \
  --eval_metrics ndcg_at_10 recall_at_100 --ignore_identical_ids True \
  --embedder_name_or_path BAAI/bge-large-en-v1.5 \
  --reranker_name_or_path BAAI/bge-reranker-large \
  --embedder_batch_size 1024 --reranker_batch_size 1024 --devices cuda:0
```

**Model 2 — bge-reranker-v2-m3** (key differences: multi-GPU, `reranker_max_length 1024`):
```bash
python -m FlagEmbedding.evaluation.beir \
  ... # same base parameters
  --reranker_name_or_path BAAI/bge-reranker-v2-m3 \
  --devices cuda:0 cuda:1 cuda:2 cuda:3 \
  --reranker_max_length 1024
```

## Comparison

import json

**Result:** `bge-reranker-v2-m3` outperforms `bge-reranker-large` on almost all metrics.
